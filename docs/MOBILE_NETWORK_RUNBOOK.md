# HamaliVpn: Wi‑Fi работает, мобильная сеть — нет

## Что это означает

Если тот же пользователь и тот же профиль работают через Wi‑Fi, сервер, UUID,
Reality-ключи и подписка в целом валидны. Отличается только путь мобильного
оператора: фильтрация IP/порта/транспорта, UDP, IPv6/APN либо клиентский DNS.

Показатель ping в Happ сам по себе ничего не доказывает: приложение может
измерять ICMP или TCP-доступность, а не передачу данных через прокси.

## Шаг 1. Проверить реальный трафик на ноде

На ноде:

```bash
curl -fsSLo /tmp/diagnose-mobile-path.sh \
  https://raw.githubusercontent.com/uclam0556-glitch/vpn/main/infra/diagnose-mobile-path.sh
sudo bash /tmp/diagnose-mobile-path.sh --capture 60
```

Во время захвата тестировщик выключает Wi‑Fi, подключает Happ и открывает сайт.

### Как читать результат

- Нет новых пакетов на `tcp/443`: мобильная сеть не доставляет соединение до
  ноды. Тестируем другой IP/регион; изменение DNS или приложения не поможет.
- Видны повторяющиеся `SYN`, но нет завершённого handshake: фильтрация
  IP/порта либо обратного трафика.
- TCP handshake проходит, затем обмен сразу останавливается: вероятна
  фильтрация транспорта/рукопожатия. Нужен резервный транспорт.
- Есть устойчивый двусторонний обмен, но сайты не открываются: проверяем DNS,
  маршрутизацию и клиентский TUN.

## Шаг 2. Проверить через прокси, а не обычным ping

В Happ выберите ping через proxy. Для централизованной настройки Happ
поддерживает:

```text
ping-type: proxy
check-url-via-proxy: https://cp.cloudflare.com/generate_204
```

Если обычный ping зелёный, а proxy ping не проходит, удалённый VPN-транспорт
не работает.

## Шаг 3. Не заменять единственный транспорт — добавить резервные

Один сервер должен выдавать несколько независимых вариантов:

1. `VLESS + Reality + XHTTP`, TCP 443 — основной.
2. `VLESS + Reality + RAW/Vision`, TCP 8443 — резерв для сетей, где XHTTP
   фильтруется.
3. `Hysteria2`, UDP 443 — резерв для сетей, где TCP ухудшается; требует
   домена и действительного TLS-сертификата.

У каждого варианта отдельный inbound, Host и понятное имя. Все inbounds должны
быть активированы на Node и добавлены во внутренний Squad.

Не ожидайте, что Hysteria2 всегда победит: некоторые мобильные сети блокируют
или ухудшают UDP. Смысл схемы — разнообразие, а не один «магический» протокол.

## Шаг 4. Клиентская фрагментация Happ

Happ официально поддерживает фрагментацию TLS ClientHello:

```text
fragmentation-enable: 1
fragmentation-packets: tlshello
fragmentation-length: 50-100
fragmentation-interval: 5
```

Сначала включайте только фрагментацию, без noises. Если она ухудшает связь,
отключите. Централизованная выдача этих параметров требует Provider ID Happ
и Response Rule/headers в Remnawave.

## Шаг 5. Второй Gcore IP

Раз баланс Gcore уже пополнен, второй короткий PAYG-тест разумен. Сначала
пробуйте Warsaw, затем Amsterdam: это другой регион и префикс, но всё ещё один
провайдер, поэтому такой тест не заменяет резерв у другого ASN.

Старую Frankfurt-ноду не удаляйте до завершения сравнения.

## Перед публичным запуском

- Минимум две ноды в разных сетях/ASN.
- На каждой минимум два транспорта.
- Автоматический proxy-health-check, а не ICMP ping.
- Резервный IP и процедура переключения.
- Reality private key, опубликованный в чате или логах, обязательно
  перегенерировать.
