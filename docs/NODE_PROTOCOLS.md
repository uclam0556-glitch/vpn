# HamaliVpn — протоколы ноды

Настраивается в **Remnawave → Профили (Xray-конфиг) + Хосты**, НЕ в Python-репо.

## Принцип

Нет одного транспорта, который гарантированно работает во всех сетях.
Мобильный оператор может отдельно фильтровать IP, TCP, UDP или конкретное
рукопожатие. Поэтому устойчивость достигается несколькими независимыми
inbounds и несколькими нодами, а не обещанием «неблокируемого» протокола.

## Что ставим на КАЖДОЙ ноде

| Протокол | Транспорт | Порт | Роль |
|---|---|---|---|
| **VLESS + Reality** | **XHTTP** | 443/tcp | Основной HTTP-транспорт |
| **VLESS + Reality** | **RAW/Vision** | 8443/tcp | Независимый TCP-резерв |
| **Hysteria2** | QUIC/UDP | 443/udp | Независимый UDP-резерв |

Все inbounds — в один сквад (`CHECHNYA-LAB`). Для автоматического выбора
используйте проверку именно через proxy, а не ICMP ping.

## Reality — ключевые параметры

- `dest` / `serverNames` (**SNI**): живой сайт с TLS 1.3, доступный с ноды и **не заблокированный в РФ**.
  Кандидаты: `www.microsoft.com`, `www.apple.com`, `dl.google.com`, крупный CDN (Cloudflare/Akamai).
- Ключи Reality (x25519) — Remnawave генерит сам (Профили).
- `flow` для VLESS+Reality+RAW/TCP Remnawave проставляет автоматически (`xtls-rprx-vision`).
- Для XHTTP используйте непустой случайный `path`. Сервер и Host должны
  выдавать одинаковый путь.

Базу JSON бери из **официальных шаблонов**: https://github.com/remnawave/templates — там готовые inbounds Reality-XHTTP и Hysteria2, меняешь только SNI.

## Правила по IP и локации (под Кавказ)

- **Чистый, незасвеченный IP.** Проверяй, что не в реестре РКН.
- Локации: **Финляндия / Прибалтика / Германия** (низкий пинг + чистые сети). Хостеры: Cherry Servers, AlexHost.
- **Турцию из РФ избегать** — плохая маршрутизация/помеченные подсети.
- Держать **1–2 резервных IP** для быстрой замены заблокированного.
- Не рекламировать сервис публично — меньше шансов засветить IP.

## Проверка (обязательно с реальной РФ-сети)

С симки МТС/Мегафон/Билайн/Tele2 (Чечня/Кавказ):
- открыть тяжёлую страницу/видео на 2–3 минуты — **не виснет ли** (тест на «заморозку 16 КБ»);
- замерить пинг, потери, скорость по обоим протоколам (см. таблицу в `NODE_TEST_PLAN.md`).

## Граница честности

Абсолютной «непойманности» нет. Устойчивость — это несколько транспортов,
несколько независимых сетей, проверка через proxy и быстрая замена отказавшего
маршрута. При режиме строгого allowlist прямой IP дата-центра может быть
недоступен независимо от настроек Xray.
